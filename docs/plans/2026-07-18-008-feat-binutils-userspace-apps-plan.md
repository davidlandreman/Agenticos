# feat: port GNU binutils as static-musl userspace applications

## Summary

Port the core GNU binutils 2.46.0 programs as separate, statically linked
musl `ET_EXEC` applications and expose them through the synthetic `/bin`
namespace. The first release contains the complete native x86-64
assemble/link/archive/inspect/transform toolchain:

```text
addr2line  ar  as  c++filt  elfedit  ld  nm
objcopy  objdump  ranlib  readelf  size  strings  strip
```

This complements TinyCC rather than replacing it. TinyCC remains the compact
on-target C compiler and continues to use its integrated assembler/linker;
GNU `as` and `ld` become standalone tools and establish the backend needed by
a future GCC driver port. The acceptance path proves that the guest can
assemble source, link an executable, execute it, create and update an
archive, and inspect or transform ELF files entirely on-target.

The port follows the committed-prebuilt model: fresh clones receive working
tools without a cross compiler or network access, while
`./userland/refresh-prebuilt.sh` rebuilds every artifact from one pinned GNU
source archive.

Upstream references:

- [GNU binutils 2.46.0 source archive](https://ftp.gnu.org/gnu/binutils/binutils-2.46.0.tar.xz)
- [GNU binutils manuals](https://sourceware.org/binutils/docs/)

## Feasibility findings

### Verified cross-build baseline

A throwaway build was completed on the project's supported Apple Silicon
development host with `x86_64-linux-musl-gcc` 14.2.0. Use the official GNU
archive and pin both fields in the Makefile:

```text
version: 2.46.0
sha256:  d75a94f4d73e7a4086f7513e67e439e8fcdcbb726ffe63f4661744e6256b2cf2
```

The working build is an out-of-tree Canadian cross:

```text
build  = aarch64-apple-darwin (derive with config.guess)
host   = x86_64-linux-musl
target = x86_64-linux-musl
```

Build only `all-binutils all-gas all-ld`; do not build GDB, gold, gprof,
gprofng, simulators, target libraries, or development libraries. Disable
NLS, debuginfod, zstd, plugins/LTO, and other optional host integrations.
The initial target set is the native x86-64/i386 ELF backend generated by
the normal `--target=x86_64-linux-musl` configuration, not a multi-arch
distribution build.

The important link detail is:

```text
CFLAGS="-Os -fno-pie"
LDFLAGS="-all-static -no-pie"
```

GNU binutils links its programs through libtool. Passing ordinary `-static`
selects static libtool libraries but still allowed a dynamically linked musl
executable in the verified prototype. Libtool's `-all-static` produced real
static `ET_EXEC` files with no `PT_INTERP`. The build must hard-fail unless
every shipped output satisfies all three conditions:

1. ELF64 x86-64;
2. `e_type == ET_EXEC`;
3. no `PT_INTERP` program header and no `DT_NEEDED` entry.

Strengthen the shared `validate_exec_elf` helper to check conditions 2 and 3
for all userland apps, so this failure mode cannot silently recur in another
port.

50 MiB Git-LFS threshold.

### Platform pieces already present

- Static musl programs, demand-paged `ET_EXEC` loading, `brk`/`mmap`,
  fork/exec/wait, signals, Linux-shaped stat calls, and large chunked file
  reads/writes are already operational.
- `/work` is a writable tmpfs-overlay directory suitable for objects,
  archives, temporary files, and executables. `/data` is writable persistent
  ext2. `/host` is read-only vvfat and already contains the TCC musl sysroot.
- The VFS supports create, truncate, seek-past-EOF, unlink, rename, hard
  links, symlinks, and validated chmod/fchmod compatibility calls.
- `readlink("/proc/self/exe")`, used by GNU `ld` to locate itself, is already
  synthesized by the userland ABI.
- The virtual `/bin` implementation already supports standalone direct ELFs
  and exposes them to `access`, `stat`, `open`, `execve`, and directory
  enumeration.

### Compatibility work to verify or add

Source inspection identified two missing Linux calls on normal write-side
paths:

1. `umask(2)` (syscall 95). BFD queries the process mask before marking a
   linker output executable. Implement a per-process mask, default `0022`,
   inherited by `fork`, retained across `execve`, and updated with normal
   `umask` return semantics. File permission enforcement remains out of
   scope, but callers must observe a coherent mask.
2. `utimensat(2)` (syscall 280). `objcopy`/`strip` use it when preserving
   dates. Support `AT_FDCWD`, explicit timestamps, `UTIME_NOW`, and
   `UTIME_OMIT` for writable paths. Add a VFS `set_times` operation for
   tmpfs, overlay, and ext2; return `EROFS` on `/host`. Nanosecond storage is
   not required where the backing filesystem only stores seconds.

Run the full guest smoke matrix with unknown-syscall tracing enabled before
declaring that list final. Do not add speculative syscall stubs. In
particular, the cross configuration intentionally leaves BFD file `mmap`
disabled, avoiding writable `MAP_SHARED` as a prerequisite; core tools use
buffered I/O instead. Plugins are disabled, so LTO `dlopen` and helper-process
paths are not acceptance requirements.

The 32-descriptor process limit is a measured risk for `ld`. Include a link
test with more than 32 input objects. BFD normally closes and reopens cached
inputs; raise the fd-table limit only if that test proves the current bound
insufficient.

## Goals

1. Reproducibly build the 14 core GNU binutils 2.46.0 applications from a
   SHA256-pinned official archive as static, non-PIE x86-64 `ET_EXEC` files.
2. Commit and stage each executable through the existing manifest-driven
   prebuilt workflow.
3. Expose every tool at its conventional `/bin/<name>` path and make zsh
   `PATH=/bin:/host` lookup work without aliases or shell hooks.
4. Configure GNU `ld` with `/host/sysroot` as its sysroot and
   `/host/sysroot/lib` as its native library search path, so `-lc` resolves
   the same musl archive used by TinyCC.
5. Implement the minimal missing ABI/filesystem semantics demonstrated by
   real guest execution, including coherent `umask` and timestamp updates.
6. Prove assemble, link, execute, archive, inspect, and rewrite workflows in
   booted QEMU tests with no unexpected syscalls.
7. Document the tool set, limitations, source pin, rebuild process, paths,
   and the handoff boundary for a future GCC port.

## Non-goals

- GCC, `cc1`, `collect2`, libgcc construction, libstdc++, or changing the
  existing `cc` alias away from TinyCC.
- GDB, gold, gprof/gprofng, `dwp`, simulators, BFD/CTF/SFrame development
  headers, shared libraries, or target runtime libraries.
- LTO plugins, `dlopen`, debuginfod, remote debug-file fetching, NLS message
  catalogs, or compressed-debug support that requires an external runtime
  library.
- Cross-target backends beyond native x86-64 plus the i386 emulations that
  the native target generates by default.
- Dynamic linking or producing executables with `PT_INTERP`. GNU `ld -static`
  and static musl output are the supported first milestone.
- Full POSIX permission ownership/enforcement. `umask` state is implemented
  for tool compatibility; chmod remains the existing validated no-op.
- A writable `/host`, installing files into a real `/usr/bin`, or replacing
  the synthetic `/bin` namespace with filesystem symlinks.

## Design

### Runtime and artifact mapping

Keep every staged host filename within FAT 8.3 constraints while exposing
normal Unix command names through the virtual namespace:

| Command | Committed artifact | Guest backing path |
|---|---|---|
| `addr2line` | `prebuilt/binutils/ADDRLINE.ELF` | `/host/ADDRLINE.ELF` |
| `ar` | `prebuilt/binutils/AR.ELF` | `/host/AR.ELF` |
| `as` | `prebuilt/binutils/AS.ELF` | `/host/AS.ELF` |
| `c++filt` | `prebuilt/binutils/CPPFILT.ELF` | `/host/CPPFILT.ELF` |
| `elfedit` | `prebuilt/binutils/ELFEDIT.ELF` | `/host/ELFEDIT.ELF` |
| `ld` | `prebuilt/binutils/LD.ELF` | `/host/LD.ELF` |
| `nm` | `prebuilt/binutils/NM.ELF` | `/host/NM.ELF` |
| `objcopy` | `prebuilt/binutils/OBJCOPY.ELF` | `/host/OBJCOPY.ELF` |
| `objdump` | `prebuilt/binutils/OBJDUMP.ELF` | `/host/OBJDUMP.ELF` |
| `ranlib` | `prebuilt/binutils/RANLIB.ELF` | `/host/RANLIB.ELF` |
| `readelf` | `prebuilt/binutils/READELF.ELF` | `/host/READELF.ELF` |
| `size` | `prebuilt/binutils/SIZE.ELF` | `/host/SIZE.ELF` |
| `strings` | `prebuilt/binutils/STRINGS.ELF` | `/host/STRINGS.ELF` |
| `strip` | `prebuilt/binutils/STRIP.ELF` | `/host/STRIP.ELF` |

`execve("/bin/readelf", ["readelf", ...])`, for example, rewrites to
`/host/READELF.ELF` while preserving `argv[0]`.

### Build recipe

Add `userland/apps/binutils/`:

```text
Makefile
README.md
.gitignore
tests/
  exit42.s
  symbols.c
patches/                 # absent unless guest evidence requires a patch
```

The Makefile must:

1. download `binutils-2.46.0.tar.xz` from GNU over HTTPS and hard-fail on the
   pinned SHA mismatch;
2. extract into a private `build/` tree and configure out-of-tree;
3. derive `--build` by running upstream `config.guess`, while setting
   `--host=x86_64-linux-musl --target=x86_64-linux-musl`;
4. pass `--with-sysroot=/host/sysroot` and
   `--with-lib-path=/host/sysroot/lib` so guest linker scripts carry usable
   search directories;
5. disable optional components/integrations and build only
   `all-binutils all-gas all-ld`;
6. copy the 14 canonical outputs (`as-new` -> `as`, `ld-new` -> `ld`,
   `nm-new` -> `nm`, `strip-new` -> `strip`, `cxxfilt` -> `c++filt`) into
   stable `build/out/` names;
7. strip them with the cross `strip`, then validate class, machine, type,
   `PT_INTERP`, and `DT_NEEDED` before publishing any output; and
8. use temporary files plus atomic rename so a failed rebuild cannot leave a
   partially refreshed executable.

Do not carry a source patch merely to make configure pass. If a patch becomes
necessary after the guest trace, keep it narrowly scoped under `patches/`,
apply it after checksum verification, and explain it in the app README.

### Manifest and prebuilt workflow

Add one `prebuilt-managed` manifest row per executable. All rows point to the
same source directory and Makefile; the first invocation performs the suite
build and later invocations are Make no-ops. This deliberately reuses the
existing staging/refresh machinery instead of adding a special bundle format.

Name rows `binutils-<tool>` so rebuild variables are unambiguous. Normal users
use the aggregate controls:

```sh
./build.sh                         # copies committed tools
./build.sh --rebuild-userland      # rebuilds all upstream apps, including binutils
./userland/refresh-prebuilt.sh     # refreshes all committed artifacts
```

Also support `REBUILD_BINUTILS=1` as a suite-level convenience. Implement it
in `stage-lib.sh` by treating it as a rebuild request for every manifest row
whose name starts with `binutils-`; do not require fourteen environment
variables.

Update `userland/prebuilt/README.md`, `userland/refresh-prebuilt.sh` comments,
and the root `CLAUDE.md` inventory in the same change as the committed ELFs.

### `/bin` namespace and the BusyBox `strings` collision

Add all 14 names to the sorted direct-app list in
`src/userland/bin_namespace.rs` and map each name to its staged host path.
Extend the existing sortedness, disjointness, rewrite, stat/access, and merged
directory tests to cover the suite.

BusyBox currently owns `strings`. There must not be two owners: disable
`CONFIG_STRINGS` in `userland/apps/busybox/busybox.config`, refresh
`BB.ELF`, and remove `strings` from `APPLETS` when GNU `strings` is added to
the direct list. The user-visible command remains `/bin/strings`; only the
implementation changes.

Keep `cc` mapped to TinyCC. Do not add prefixed aliases such as
`x86_64-linux-musl-ld` in the first milestone; the conventional unprefixed
commands are sufficient inside this single-target OS.

### `umask` and timestamp semantics

Add `umask: u32` to each ring-3 process. Initialize it to `0o022`, clone it on
`fork`, preserve it on `execve`, and implement syscall 95 as:

```text
old = process.umask
process.umask = requested & 0o777
return old
```

The filesystem continues to synthesize/no-op most permission bits, but BFD
now gets coherent answers and future permission enforcement has the correct
state owner.

Add `Filesystem::set_times(path, atime, mtime)` and VFS routing. Tmpfs stores
seconds on its nodes; overlay routes upper files and copies up lower files
only when mutation is legal; ext2 updates inode atime/mtime plus ctime; FAT
and vvfat return read-only/unsupported errors as appropriate. Implement the
Linux x86-64 `utimensat` ABI with bounded path copy-in, `AT_FDCWD`, flags
validation, and the `UTIME_NOW`/`UTIME_OMIT` sentinel values. Tests must cover
explicit values, NOW/OMIT, bad flags/pointers, missing paths, and `/host`
read-only behavior.

### Guest compatibility audit

Before fixing any additional ABI, stage the built tools and run each one in
QEMU with unknown-syscall tracing enabled. Exercise both read-only and
write-side modes, record every `ENOSYS`, and classify it as:

- required and implement correctly;
- an optional feature probe with a working fallback; or
- unreachable because the associated feature is intentionally disabled.

The merge gate is no unknown syscall on the acceptance workflows. Optional
`--help`/`--version` code may probe harmlessly only if the exact call and
fallback are documented in the app README.

## Risks and mitigations

| Risk | Impact | Mitigation |
|---|---|---|
| Libtool quietly emits dynamic executables | loader rejects every tool | require `-all-static`; validate program headers and dynamic tags |
| Fourteen prebuilts enlarge the repo by 15.5 MiB | clone/checkout growth | use `-Os`, strip, measure; switch to a bundle/LFS only at documented threshold |
| BusyBox and GNU both claim `strings` | ambiguous dispatch/duplicate `/bin` entry | disable BusyBox applet in the same unit that adds GNU `strings` |
| `ld` default paths point at host-build prefixes | `-lc` and CRT lookup fail | compile `/host/sysroot` sysroot and library paths in; assert verbose search dirs |
| Binutils optional features pull in subprocesses/plugins | unsupported `dlopen`, LTO, extra syscalls | disable plugins, GDB, profilers, debuginfod, NLS, and non-scoped components |
| Write-side tools expose syscall gaps | archive/objcopy/strip failures | trace real guest workflows; implement `umask`/`utimensat`; no speculative stubs |
| 32-fd table is too small for a real link | larger programs fail | test >32 inputs and raise the limit only on measured failure |
| vvfat read latency for many inputs | slow large links | write outputs in `/work`; keep scope functional; profile/cache in a follow-up |
| `elfedit` wants writable shared mmap for some modes | selected header edits fail | build without BFD file-mmap, test supported buffered modes, document advanced property edits as deferred if needed |

## Completion criteria

The feature is complete when all 14 conventional commands resolve from zsh,
their committed artifacts are static and reproducible, GNU `as` plus GNU `ld`
can create and execute a fresh guest binary, archive and ELF transformation
work on `/work`, `strings` has one namespace owner, the traced acceptance
matrix has no unexpected syscalls, and both focused and full QEMU suites pass
without requiring network access or a host cross toolchain.

## Implementation result

Completed on 2026-07-18. The fourteen stripped artifacts total 16,245,912
bytes. Traced guest execution additionally demonstrated that GNU `as` needs
`readv(2)` and GNU BFD needs accurate writable-file `F_GETFL`; both were
implemented alongside the planned `umask(2)` and `utimensat(2)` work.

Validation completed with `make -C userland/apps/binutils validate`, an
`ET_EXEC` + `PT_INTERP` negative validator fixture, `cargo fmt --check`,
`cargo check --features test`, a normal `./build.sh -n`, the focused binutils
and ABI/filesystem filters, and the complete booted suite: all 885 tests pass.
The binutils module also passes with `AGENTICOS_TEST_NETWORK=off`.
